# 在项目上的一些思考

## RAG 的使用与否问题
### 项目的流程
```text
用户给出需求
→ Agent 规划
→ 工具调用（代码生成 / 搜索 / 构建）
→ 产出完整可运行项目
→ 返回
```
**_这叫 Tool-augmented Agent，不是 RAG Agent。_**
### 暂时不做 RAG
**现在不做 RAG，但：**
- 设计接口时，给 Retrieval 留一个 Hook
- Tool 体系里，留一个 KnowledgeTool 空位
- Prompt 结构别写死
> 这样未来加 RAG，只是：“多接一根管子”
> 
> 而不是：“拆房子重盖”
