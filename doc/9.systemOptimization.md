## 系统优化

### 性能优化

#### 并发模型

>现在的瓶颈本质不是 Java，而是 LLM 调用 + Agent 串行协作。

**核心策略：**
- Agent 级并发（不是 Tool 级） 
- 阶段并发 + 有序汇总

#### 缓存策略

可以缓存什么？

| 类型          | 建议           |
| ----------- | ------------ |
| Prompt 模板   | 强缓存          |
| 工程结构扫描结果    | 短期缓存         |
| 历史 Agent 结论 | 基于 hash 的软缓存 |
| 用户最近一次生成结果  | session 缓存   |


### 安全性优化

#### Prompt 安全

**双层防御：**

- 用户 Prompt 预审 
- Agent 自检 Prompt

示例规则

- 禁止：
    - 删除非白名单目录 
    - 执行 shell / mv 根目录 
    - 覆盖 .git / .env

#### 流量与滥用保护

- 用户级：
    - QPS
    - Token 上限
- 项目级：
    - 并发生成任务数
- Agent 级：
    - 单任务最大思考轮次

### 稳定性优化

**推荐策略：**

| 场景         | 策略       |
| ---------- | -------- |
| LLM 超时     | 指数退避     |
| Tool IO 失败 | 有限重试     |
| Agent 推理异常 | 降级 Agent |


### 可观测性

你需要：
- 每次生成任务：
    - Agent 执行链路
    - Tool 调用时间
    - Token 消耗
- 可视化 Dashboard

> 没有可观测性，优化就是玄学。
