# 系统优化模块设计文档
> xiaorui，每个项目都必须做的系统优化部分。

## 性能优化
### 并发模型

> 现在的瓶颈本质不是 Java，而是 LLM 调用 + Agent 串行协作。

**核心策略：**
- Agent 级并发（不是 Tool 级） 
- 阶段并发 + 有序汇总

#### 设计实现

自己是设计一个 agent 编排器 `DefaultAgentOrchestrator`，其实有点类似与执行工作流。
在用户发送信息与 agent 对话后，先保存 agent 的执行状态，而不是等待 agent 返回完整结果。
将 agent 执行到最终的回复作为一个个任务串起来，提交任务后立即返回状态，
然后根据状态获取执行结果，避免了接口阻塞等待。

### 失败重试

可以重试的失败（SYSTEM_RETRYABLE 系统失败）
- LLM 超时
- 网络异常
- Redis / MySQL 短暂不可用
- Agent 执行被中断

### 缓存策略

> 对于主页精选应用进行缓存，使用的是 @Cacheable 注解进行 Redis 缓存。

### 流量与滥用保护

- 用户级：
    - QPS
    - Token 上限
- 项目级：
    - 并发生成任务数
- Agent 级：
    - 单任务最大思考轮次

> 最终项目中使用的 Redisson 的 RateLimiter 来实现的接口、用户和 API 级别的限流。

## 稳定性优化

**推荐策略：**

| 场景         | 策略       |
| ---------- | -------- |
| LLM 超时     | 指数退避     |
| Tool IO 失败 | 有限重试     |
| Agent 推理异常 | 降级 Agent |
