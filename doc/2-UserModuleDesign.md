# 用户模块设计文档
> xiaorui，用户模块这部分其实基本上都是自己做一些项目后总结出来的自己的一套体系，然后参考一些开源项目的实现思路，不断优化用户模块。

## 总览
相关用户模块的常见功能实现，如注册登录、权限校验、用户信息管理等，
考虑到管理员的同等用户功能需求，所以只是在平台上增加管理员权限功能，未分离普通用户和管理员角色的系统。
用户模块较难实现（或者说需要深度优化）的功能是“用户登录和 token 管理”，这部分是应该不断优化和完善的。

## 设计思路
> 其实主要还是根据以往项目的经验总结和优化，以及通过学习其他开源项目的实现思路，来设计和实现用户模块。

### 库表设计
**参考 `db/create_table.sql`。**
> 但其实还是有些字段未去使用，尽量避免过度设计，按需增加字段即可。

### 注册登录
1. **采用邮箱注册登录**：邮箱验证码注册登录，并使用 Hutool 工具库生成图片验证码，多加一重验证防护，验证码等都设置过期时间存储在 Redis 中
2. **登录信息传输防护**：原计划是前端在传输时使用 AES 加密，后端先使用 AES 解密，再使用 BCrypt 加密，但是目前为了方便测试，前端暂时不做加密处理
3. **登录信息检查防刷**：使用 Redis 的 `incr` 原子操作记录登录次数，解决高并发 `count++` 的非原子问题，超过一定次数则锁定账号
4. **用户登录信息存储**：将用户信息封装存储到以 Sa-Token 技术生成的 token 中，设置过期时间并存储在 Redis 中

### 权限校验
1. **自定义权限校验注解**：由于是比较通用的业务需求，所以将其从业务代码中分离出来，使用 AOP 切面 + 自定义注解的方式实现。
2. **数据库权限字段区分**：在库表设计上用 `user_role` 字段区分用户角色，普通用户和管理员，普通用户为 `user`，管理员为 `admin`
> 当然，如果有更复杂的业务需求，也可以使用 Spring Security、Sa-Token 等框架实现权限的精细控制，但目前阶段项目暂时没有需求使用。

### 信息管理
1. **用户信息管理**：用户基本的信息管理（CURD），以及对应的管理员权限功能的信息管理，此外还扩展了管理员批量删除用户、封禁或解封用户
2. **头像文件存储**：区别与之前学习使用的云存储（OSS、COS、七牛云等），本次项目使用本地 Linux 系统搭建的 MinIO 对象存储服务，实现文件的存储
> 管理员功能后续还会新增扩展功能，如用户信息导入导出、系统日志导出等等（以往项目的功能，后续对该项目扩展时都会陆续添加）
