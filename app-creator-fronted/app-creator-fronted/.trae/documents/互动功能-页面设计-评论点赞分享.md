# 互动功能页面设计（桌面端优先）

## 全局设计约束（适用于所有页面）

### Layout
- 桌面端优先：内容容器建议 `max-width: 1200px`，左右留白自适应；主要分区采用 Flexbox（双栏/工具条）与 Grid（卡片列表）混合。
- 响应式策略：
  - >= 1200px：双栏/多列卡片
  - 768px ~ 1199px：卡片降列、按钮改为图标+Tooltip
  - < 768px：工具条换行、评论区输入改为全宽

### Meta Information
- 基础：`title` 随路由变化（主页/应用对话/登录注册）。
- 分享（Open Graph）：因是 SPA，可在后端渲染或通过预渲染支持；本次前端仅保证页面内“复制链接/分享记录”能力。

### Global Styles（Design Tokens 建议）
- 背景：`#0b1220` ~ `#111827` 渐变/暗色基底（与现有首页风格一致）。
- 主色：橙色系（例如 `#F28C28`）用于主按钮与高亮。
- 文本：正文 `#E5E7EB`（暗底），次级 `rgba(229,231,235,0.7)`。
- 卡片：白色半透明底 `rgba(255,255,255,0.95)` + 轻模糊 `backdrop-filter: blur(10px)`。
- 互动按钮：
  - 默认：灰色图标/文字
  - 激活（已点赞/已分享）：主色填充或主色描边
  - Hover：提升对比度 + 轻微上浮（与现有 AppCard hover 一致）

## 页面 1：主页（/）

### Layout
- 页面主体使用“Hero + 两个卡片区块（我的应用/精选应用）”的纵向堆叠布局。
- 卡片列表区使用 CSS Grid：`grid-template-columns: repeat(3, 1fr)`（桌面端），间距 `24px`。

### Page Structure
1. 顶部 Hero（保持现有）
2. 应用列表区块
   - 标题/更多
   - AppCard 网格

### Sections & Components

#### AppCard（新增：点赞/分享入口，方案1）
**位置与层级**
- 位于卡片信息区（`.app-info-right`）底部，紧跟作者信息后方。
- 采用 `a-space` 水平排列两个轻量按钮：点赞、分享。

**元素**
- 点赞按钮：图标 + 计数（`app.likeCount`）
- 分享按钮：图标 + 计数（`app.shareCount`）
- Tooltip：
  - 未登录：提示“请先登录后再点赞/分享”
  - 已登录：提示“点赞/取消点赞”“分享/取消分享”

**交互与状态**
- 点击后立即进入 loading（按钮局部 spinner 或 disabled）。
- 成功：
  - 切换激活样式（已点赞/已分享）
  - 本地计数 +1 / -1（或重新拉取 AppVO 以对齐后端）。
- 失败：展示 message 提示，不改变激活态。

## 页面 2：应用对话页（/app/chat/:id）

### Layout
- 顶部固定高度工具条（现有 `.header-bar`）+ 下方左右分栏（对话区/预览区）。
- 工具条右侧区域（`.header-right`）增加互动按钮组。

### Meta Information
- Title：`{appName} - 应用对话`
- Description：展示应用对话与预览。

### Sections & Components

#### 2.1 详情头部互动区（新增，方案1）
**位置**
- `.header-right` 内：建议放在“应用详情”按钮旁边（优先靠左），避免与“下载/部署”主流程按钮抢占视觉焦点。

**元素**
- 点赞图标按钮 + 计数
- 分享图标按钮 + 计数

**交互**
- 未登录：点击触发登录引导（跳转登录页或弹 message 并跳转）。
- 已登录：调用对应 controller 完成 toggle。

#### 2.2 AppDetailModal（应用详情弹窗，新增评论 Tab，方案1）

**结构**
- 弹窗宽度保持 `500px`（现有）。
- 顶部内容由“单列信息”升级为 `a-tabs`：
  - Tab1：基础信息（保留现有信息块与操作栏）
  - Tab2：评论（新增）

**评论 Tab 设计**
- 顶部：评论输入区
  - `a-textarea`（2~4 行自适应）
  - 右下：提交按钮（主色），未登录禁用并提示登录
- 中部：评论列表
  - 单条评论：头像、昵称、时间（次级文字）、内容
  - 右侧操作：仅评论作者显示“删除”按钮（danger link）
- 底部：分页/加载更多
  - 简化方案：加载更多按钮（按页拉取 `/appComment/query`）

**空/错状态**
- 空：`a-empty`，文案“还没有评论，来抢沙发吧”
- 错：`a-alert` + 重试按钮

## 页面 3：用户登录/注册页（/user/login、/user/register）

### Layout
- 单列居中卡片布局（Form + 辅助链接），桌面端卡片宽度建议 `420~480px`。

### Meta Information
- Title：登录页“用户登录”，注册页“用户注册”。

### Sections & Components

#### 登录后返回（为互动操作服务）
- 若从点赞/分享/评论跳转而来，登录页读取 `redirect` query（例如 `?redirect=/app/chat/123`），登录成功后跳回。
- 若无 redirect，默认回主页。

#### 交互状态
- 表单提交 loading、错误提示沿用现有 message/toast 体系。
