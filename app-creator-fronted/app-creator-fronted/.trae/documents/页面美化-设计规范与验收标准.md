## 1. Global Styles（桌面优先）
### 1.1 设计目标
- 统一“品牌主色 + 中性灰阶 + 状态色”体系，减少页面间割裂。
- 统一 8px 间距体系、卡片圆角、阴影层级、表单密度。
- 保持 Ant Design Vue 交互习惯（focus ring、disabled、loading）。

### 1.2 建议设计令牌（基于当前代码中的既有颜色，若与你约束冲突，以你约束为准）
- Brand / Primary：#1890ff（导航标题、用户消息气泡已使用）
- Brand / Gradient（首页已使用）：#3b82f6 → #8b5cf6 → #10b981
- Neutral：#0f172a / #1e293b / #64748b / #94a3b8 / #e2e8f0 / #f8fafc
- Status：Success #52c41a、Warning #faad14、Danger #ff4d4f（与 AntD 一致）
- Radius：8（卡片）/ 12（气泡）/ 16（大输入框）
- Shadow：
  - S：0 2px 8px rgba(0,0,0,.08)
  - M：0 10px 40px rgba(0,0,0,.12)

### 1.3 排版
- 字体：沿用系统字体栈（当前全局已设置）。
- 标题层级：
  - H1 24–32 / SemiBold；H2 18–20 / SemiBold；正文 14–16。
- 行高：正文 1.6；表单描述 1.4。

## 3. Page Design Specs

### 3.1 全局布局与导航（BasicLayout / GlobalHeader / GlobalFooter）
- Layout：Header + Content + Footer；内容区默认 1200px max-width，左右 padding 20–24；使用 Flex + Ant Grid。
- Meta：全站 title 模板 `鱼皮 AI 应用生成 - {页面名}`；description 复用一句话卖点。
- Header：
  - 左侧 Logo+站名基线对齐；站名使用品牌色（若你禁用渐变则保持纯色）。
  - 菜单：hover/selected 状态统一；禁止出现底部多余边框（当前已去除）。
  - 用户区：未登录为主按钮“登录”；已登录为头像+昵称，下拉菜单两项。
- Footer：弱化背景与分隔线，文本使用 Neutral 500。

### 3.2 首页（HomePage）
- Layout：纵向分区（Hero → 输入区 → 快捷按钮 → 我的作品 → 精选案例）。容器居中，卡片网格 CSS Grid。
- Hero：
  - 标题使用渐变文字可保留；需要保证在亮底上对比清晰（必要时加轻微 text-shadow）。
  - 描述文案保持语义不变；字重与字色弱化。
- 输入区：
  - 输入框为核心 CTA 组件：focus 时提升阴影与边框/外发光（与 AntD focus 一致）。
  - 发送按钮需具备可点击面积≥40px，高对比主色。
- 快捷按钮：一行不足则自动换行；按钮采用“轻底 + 细描边 + hover 上浮”。
- 卡片区：
  - 网格最小列宽 320；卡片视觉统一：封面比例一致、标题两行截断、操作按钮位置统一。
  - 分页居中，信息“共 X 个应用/案例”弱化。

### 3.3 登录 / 注册 / 重置密码
- Layout：居中表单卡片（max-width 720），上方标题+副标题。
- 表单密度：FormItem 间距统一（建议 16）；错误提示不跳动（预留高度）。
- 验证码区：图形验证码图片高度统一为 56；“刷新验证码”按钮为次按钮。
- 链接：右对齐但不拥挤；hover 下划线或颜色变化。

### 3.4 应用对话页（AppChatPage）
- Layout：桌面端左右分栏（对话 40% / 预览 60%）；≤1024px 改为上下分区。
- 顶部栏：左侧“应用名 + 生成类型 Tag”；右侧按钮三段分组：信息（详情）/ 资产（下载）/ 动作（部署）。
- 对话区：
  - 消息容器滚动：滚动条样式轻量；“加载更多历史消息”为弱入口。
  - 气泡：用户=主色底白字；AI=浅灰底深字；Markdown 容器与代码块背景、圆角、内边距一致。
  - 输入区：固定在对话区底部；发送按钮固定在右下角；禁用态要有明确提示（已有 tooltip）。
- 选中元素提示条：信息按“标签/选择器/内容/页面路径”分组，code 使用等宽字体和浅底。
- 预览区：
  - 三态：无预览占位 / 加载中 / 错误；每态都提供“新窗口打开”入口（符合现有行为）。
  - iframe 边界：与卡片圆角一致，避免溢出。

### 3.5 编辑应用页（AppEditPage）
- Layout：单列内容，max-width 1000；标题区固定层级。
- 表单卡片：纵向布局，字段说明（form-tip）统一弱化颜色；封面预览块为信息卡样式。
- 应用信息：Descriptions 分组清晰，label 背景弱化且与卡片头统一。

## 4. 验收标准（首批必须通过）
### 4.1 不回归（硬性）
- 不新增/删除任何页面入口与权限逻辑（菜单过滤、按钮禁用、跳转保持一致）。
- 首页创建应用、登录/注册/验证码、对话生成/预览/部署/下载、编辑应用保存流程均可用。

### 4.2 视觉一致性（硬性）
- 首批页面统一：主色、圆角、阴影、间距体系、标题层级、按钮主次关系。
- 状态覆盖：hover / active / focus / disabled / loading / empty / error 在首批页面均可见且一致。

### 4.3 布局与响应式（硬性）
- 桌面 1440/1280/1024 与 768 下：无横向滚动；关键区域（Header、主要按钮、表单、对话与预览）不遮挡不溢出。

### 4.4 可读性与无障碍（建议但默认验收）
- 正文对比度达到可读水平；弱提示不低于 Neutral 400；重要 CTA 不使用仅靠颜色区分。
